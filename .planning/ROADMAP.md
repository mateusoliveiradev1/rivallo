# Roadmap: Rivallo

All phases are sequential. Gate approval is a hard boundary.

| Phase | Gate | Outcome |
|---:|---|---|
| 1 | 0 | Product, architecture, data, test, design, and ADR planning approved |
| 2 | 1 | Workspace/toolchains and quality scripts established |
| 3 | 1 | Rust modular-monolith crates and API contract pipeline established |
| 4 | 1 | Desktop shell, local persistence boundary, containers, CI skeleton; minimum builds pass |
| 5 | 2 | Token source, icon policy, UI primitives, and UI Lab designed/implemented |
| 6 | 2 | Approved briefs for bootstrap, club selection, dashboard, squad; visual QA passes |
| 7 | 3 | Dev identity, fixtures, API health/session/clubs/context endpoints |
| 8 | 3 | Club selection and SQLite-backed restore through application ports |
| 9 | 3 | Dashboard, squad, cache/offline states, sync indicator |
| 10 | 3 | End-to-end, accessibility, visual, migration, contract and release evidence |
| 11 | V0.2 | Auth decision and private-league creation/invite |
| 12 | V0.2 | Authoritative readiness/round advance and realtime updates |
| 13 | V0.2 | Reconnection, Neon persistence, two-desktop proof |

## Gate criteria

- **Gate 0:** documentation approved; no production code.
- **Gate 1:** all minimal builds and CI jobs pass; review approval required.
- **Gate 2:** approved screen briefs, UI Lab, visual and accessibility evidence.
- **Gate 3:** V0.1 flow works online/offline with evidence; review approval required.
