<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| PATH executable to validator | Version output from a developer-controlled executable is untrusted input. |
| Native policy files to validator | Repository policy controls whether a local environment may run quality checks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-03 | Spoofing | PATH version commands | mitigate | Parse constrained stable output and execute commands via argument arrays, not a shell. |
| T-02-04 | Tampering | Toolchain policy | mitigate | Read policy from versioned manifests and test below-minimum/mismatch outcomes. |
| T-02-05 | Elevation of privilege | Validator / rustup child process | mitigate | Do not call package managers, rustup, installers, or update commands; force RUSTUP_AUTO_INSTALL=0 for each Rust/Cargo probe and give manual remediation. |
| T-02-SC | Tampering | Tooling test fixtures | mitigate | Use temporary, isolated fixtures and cleanup assertions. |
</threat_model>
