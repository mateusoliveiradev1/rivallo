<files>crates/platform/Cargo.toml, crates/platform/src/lib.rs, crates/platform/src/runtime.rs</files>
<behavior>
- `GET /health` reports liveness without product state.
- `GET /ready` returns a bounded typed response containing the fixed service identity, the contracts-owned version, and a neutral runtime protocol marker.
- A response is reusable only when every expected field exactly matches; malformed, unhealthy, and incompatible responses are diagnostics rather than ready state.
- The listener binds `127.0.0.1` at the fixed planned port and supports cancellation-driven graceful shutdown.
- The sidecar reads its private stdin shutdown-control message and cancels that same server token; unrelated stdin data cannot expose a network management surface.
</behavior>
<action>Add only `axum` 0.8.9 and the narrowly featured `tokio` 1.52.3 runtime dependencies to the outer platform crate; preserve platform's existing dependencies on application/contracts and do not add either framework to domain or application. Test the runtime contract before implementation, then create `runtime.rs` with constants for the fixed loopback address, five-second readiness ceiling, short polling cadence, service identifier, and protocol marker. Implement local `/health` and `/ready` routes, a typed readiness payload derived from `rivallo-contracts::CONTRACT_VERSION`, strict response validation for desktop reuse, and cancellation-token graceful server composition. Define a private shutdown-control reader over sidecar stdin: it recognizes the fixed `shutdown` control message and invokes the exact cancellation token passed to the server; prove with Rust tests that the signal ends the graceful server path and that unrelated input does not cancel it. Keep this control channel private to the managed child stdin, with no HTTP management route or user-provided command input. Keep routes free of product state, authentication, persistence, public-network binding, mutable port selection, shell commands, and raw error presentation. Re-export only the runtime identifiers required by the sidecar binary and later host integration.</action>
