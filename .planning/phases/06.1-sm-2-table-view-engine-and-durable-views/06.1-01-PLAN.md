## Trust Boundaries

| Boundary | Description |
|---|---|
| Decoded repository/IPC state → pure engine | Locally persisted JSON is untrusted until every ID, bound, capability, and typed value is validated. |
| UI intent → accepted durable proposal | Pointer/keyboard commands may be stale or invalid and must not partially mutate the last valid state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-061-01-T | Tampering | `applyTableViewCommand` / schema validation | mitigate | Exhaustive typed validation, immutable reduction, concrete rejection events, and prior-state equality tests. |
| T-061-01-D | Denial of Service | widths, pins, filters, sorts, collections | mitigate | Finite width clamps plus explicit pin, depth, clause, sort, page, and collection bounds are tested before any render work. |
| T-061-01-S | Spoofing | stable table/view/column/row identities | mitigate | Accept only owning-schema semantic IDs; reject labels, indexes, duplicate IDs, and unknown IDs as authority. |
| T-061-01-I | Information Disclosure | pure engine inputs | accept | The engine receives committed fictional/local table intent only and performs no I/O, logging, network, or secret access. |
</threat_model>
