## Decisions Made

- Fixed required start pins are explicitly locked in schema metadata, while `position` and optional columns retain configurable pin capability.
- Aggregate pin validation uses the owning schema's finite default visible-width basis, so hiding optional columns does not make a previously valid locked-pin layout invalid while count and 50% bounds remain enforced.
- Ordered football descriptors reuse the existing pure squad comparator; stable name ordering is the final deterministic tie-break.
- SM-02 is addressed but not marked complete because the requirement spans all eight Phase 06.1 plans.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Corrected aggregate pin-width validation for reduced-column views**

- **Found during:** Task 2 GREEN, while constructing `Mostrar somente gols`
- **Issue:** Comparing pinned width with only the currently visible column width caused the required locked start columns to invalidate the canonical required-columns-plus-goals view.
- **Fix:** Compare aggregate pinned width with the owning schema's finite default visible-width basis and strengthen the engine regression fixture to exceed that bound explicitly.
- **Files modified:** `apps/desktop/src/table-view/table-view-engine.ts`, `apps/desktop/src/table-view/table-view-engine.test.ts`
- **Verification:** Engine and scorers-view pipeline suites pass.
- **Committed in:** `c7c0f9f`

**2. [Rule 2 - Missing critical functionality] Distinguished locked default pins from configurable pinning**

- **Found during:** Task 2 GREEN, while encoding the exact UI-SPEC supported operations
- **Issue:** A boolean pinnable capability alone could not express the three required columns that start pinned but do not expose pin/unpin operations.
- **Fix:** Added schema-owned `pinningLocked` validation and command rejection without adding UI or persistence authority.
- **Files modified:** `apps/desktop/src/table-view/table-view-engine.ts`, `apps/desktop/src/matchday/squad-table-schema.ts`
- **Verification:** Exact capability tests and all command-validation regressions pass.
- **Committed in:** `c7c0f9f`

**Total deviations:** 2 auto-fixed (1 bug, 1 missing critical contract distinction).
