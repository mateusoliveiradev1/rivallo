## Decisions Made

- The application owns an exact private mirror of the current `squad.primary` system baseline rather than trusting an arbitrary caller-provided system view.
- Provenance and mutability are separate closed values whose valid pairings are enforced before any state can be saved.
- General save proposals may change only user-owned views; system-default and shared-read-only views must remain byte-for-byte equal to the authoritative prior state.
- A failed import returns the prior durable state and a retryable proposal without a receipt; only a successful atomic save returns a receipt.
- `ownerScope: local-fixed` remains an explicit enum seam for later Phase 9 ownership migration without adding identity, SQLite, query, cache, queue, or synchronization behavior.
