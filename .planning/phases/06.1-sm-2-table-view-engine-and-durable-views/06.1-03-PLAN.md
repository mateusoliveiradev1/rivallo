<threat_model>

## Trust Boundaries

| Boundary | Description |
|---|---|
| App-data filesystem → platform decoder | Files can be corrupt, truncated, stale, oversized, tampered, or written by a newer application version. |
| Valid in-memory repository state → replacement transaction | Process interruption or Windows rename semantics can otherwise create loss between old and new generations. |
| Quarantine evidence → diagnostics | Recovery must retain useful bounded evidence without disclosing full local view/player payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-061-03-T | Tampering | storage decoder and migration chain | mitigate | Byte bounds, exact envelope/schema/owner checks, adjacent-only migrations, full application validation, and whole-payload quarantine. |
| T-061-03-D | Denial of Service | JSON bytes, collections, migration traversal | mitigate | Reject oversized files/collections before allocation and cap migration steps to the registered adjacent chain. |
| T-061-03-R | Repudiation | recovery/quarantine outcome | mitigate | Persist bounded reason code, source version, safe fingerprint, and transaction outcome without full-payload logging. |
| T-061-03-I | Information Disclosure | quarantine files/logs | mitigate | Keep quarantine under host-owned app-data, never accept UI paths, and log only bounded identifiers/reason codes. |
| T-061-03-E | Elevation of Privilege | filesystem path selection | mitigate | Tauri injects one fixed path; view/table IDs are data and never path segments. |
</threat_model>
