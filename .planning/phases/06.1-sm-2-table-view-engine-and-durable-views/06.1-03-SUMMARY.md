## Decisions Made

- Storage-envelope versioning is platform-owned and currently version 3; the serialized application state retains its application-owned metadata version 1.
- Historical storage v1 removes only `removedColumn`, already documented by the existing legacy preference contract. Any other unknown ID remains incompatible and quarantines the whole payload.
- Storage v2 adds `averageRating` by reading the exact column state from `squad_system_default_repository_state`, preserving every recognized older view field byte-for-byte after typed decode.
- Recovery precedence is deterministic: a valid active generation wins; without one, a complete staged generation wins; otherwise a valid backup is restored.
- Quarantine records bounded reason code, source versions, candidate class, FNV-1a fingerprint, original/diagnostic byte counts, truncation state, and separate raw diagnostic payload without logging local view contents.

- **Found during:** Task 1 GREEN
- **Issue:** On Windows, `File::open` creates a read-only handle, so `sync_all` failed after copying the valid active generation to backup.
- **Fix:** Reopened the copied backup with explicit read/write permissions before `sync_all`.
- **Files modified:** `crates/platform/src/table_view.rs`
- **Verification:** Deterministic repeat-save and all interruption tests pass natively on Windows.
- **Committed in:** `4ba74a1`
