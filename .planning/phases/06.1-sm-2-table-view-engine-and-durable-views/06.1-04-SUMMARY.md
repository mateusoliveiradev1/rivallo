## Decisions Made

- The desktop contract uses the already-built engine vocabulary so the next screen controller can consume decoded state directly; only the host translates to filesystem-neutral application types.
- Load seeding is exposed as ordinary `loaded`, while migrated, recovered, unavailable, invalid, and initial save-failure outcomes remain explicit textual controller states.
- Save confirmation carries the exact accepted repository revision; legacy confirmation carries the application-owned source-version/fingerprint receipt and must also exist in the decoded durable state.
- v4 owns current optional-column visibility, while pre-column v2/v3 payloads materialize `averageRating` from `SQUAD_TABLE_SCHEMA`; recognized legacy entries retain their relative order and visibility.
