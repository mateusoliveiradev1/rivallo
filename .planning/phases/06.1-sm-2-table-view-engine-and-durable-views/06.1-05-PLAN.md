<threat_model>

## Trust Boundaries

| Boundary | Description |
|---|---|
| Runtime-decoded client outcome → React controller | Valid DTO shape may still represent failure, recovery, stale request completion, or incompatible lifecycle intent. |
| User interaction → async persistence confirmation | Rapid commands, navigation, and StrictMode effects can race and must not overwrite a newer proposal/baseline. |
| Legacy adapter result → retirement call | Browser table fields must survive until the exact Rust receipt is confirmed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-061-05-T | Tampering | controller proposal/baseline | mitigate | All proposals pass the pure reducer; baseline advances only from decoded confirmed repository responses. |
| T-061-05-R | Repudiation | async save/import result | mitigate | Request identity plus typed confirmed receipt/outcome lets stale completions be ignored and failure remain visibly dirty. |
| T-061-05-D | Denial of Service | repeated clicks/StrictMode effects | mitigate | One in-flight mutation per intent, idempotent load/import, disabled repeat submission, and bounded engine state. |
| T-061-05-E | Elevation of Privilege | React storage/repository access | mitigate | Components/hooks import only engine and typed client/legacy APIs; the compatibility adapter is the sole browser-storage exception. |
| T-061-05-S | Spoofing | system/read-only provenance | mitigate | Controller renders capabilities from validated provenance and cannot expose forbidden mutation callbacks. |
</threat_model>
