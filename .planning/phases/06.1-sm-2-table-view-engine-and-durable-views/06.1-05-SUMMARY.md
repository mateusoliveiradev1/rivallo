## Decisions Made

- Repository state remains usable through unavailable, corrupt, future-version, migrated, recovered, command-rejection, and save-failure outcomes; React reports typed product state rather than inventing network wording.
- Dirty system or read-only proposals remain previewable and are saved later through provenance-valid lifecycle operations rather than mutating immutable views.
- Elenco-to-Táticas navigation consults the explicit controller guard whenever the table proposal is dirty; clean navigation and gameplay actions remain immediate.
- The existing `rivallo.squad-ui` compatibility keys remain the DATA-01 seam for non-table preferences, but `UiPreferences` and its writer no longer contain density or visible-column fields.
