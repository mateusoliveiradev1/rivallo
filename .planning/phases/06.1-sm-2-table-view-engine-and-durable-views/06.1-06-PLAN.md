<threat_model>

## Trust Boundaries

| Boundary | Description |
|---|---|
| Validated controller capabilities → rendered lifecycle actions | UI must not accidentally expose rename/delete/save authority for immutable provenance. |
| Dirty user proposal → destructive navigation/delete | A dismiss, view switch, or screen switch could silently discard unsaved intent without a guarded decision. |
| Repository outcome → status/announcement | Recovery and failure copy must be truthful and must not misrepresent local repository failure as network state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-061-06-S | Spoofing | `SavedViewSelector` provenance actions | mitigate | Render actions solely from closed validated provenance/capabilities and test forbidden actions are absent. |
| T-061-06-T | Tampering | dirty transition dialogs | mitigate | Central save/discard/continue guard, confirmed baseline updates, stable IDs, and no implicit commit/discard. |
| T-061-06-R | Repudiation | save/delete/default feedback | mitigate | Exact view/fallback names in visible status and live announcements tied to confirmed controller outcomes. |
| T-061-06-D | Denial of Service | repeated lifecycle submissions/focus loops | mitigate | Busy disables duplicate mutation, bounded names/lists, Escape paths, deterministic focus return, and repeated-cycle tests. |
| T-061-06-I | Information Disclosure | repository errors | mitigate | Present bounded product-safe reason/copy only; never show app-data paths, raw payload, quarantine bytes, or process details. |
</threat_model>
