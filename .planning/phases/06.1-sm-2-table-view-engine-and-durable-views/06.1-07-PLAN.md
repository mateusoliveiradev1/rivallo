## Trust Boundaries

| Boundary | Description |
|---|---|
| Pointer/keyboard operation session → engine command | Drag coordinates, repeated keys, stale column IDs, and invalid geometry are untrusted transient intent. |
| Normalized table state → CSS geometry | Finite validated values must not become intrinsic amplification, clipped focus, or overlapping sticky regions. |
| Filter/sort descriptor → football rows | Generic descriptors may be unsupported or stale and must not alter XI/gameplay identity or invent domain semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-061-07-T | Tampering | customizer/table commands | mitigate | All pointer/keyboard changes dispatch the same validated engine command; rejected events retain prior state. |
| T-061-07-D | Denial of Service | widths/pinning/filter rendering | mitigate | Finite schema bounds, four/50% pin caps, bounded clauses, one overflow owner, and four-viewport geometry tests. |
| T-061-07-S | Spoofing | required/read-only/unsupported capabilities | mitigate | Controls derive from validated schema/provenance; required reasons render and unsupported grouping controls are absent. |
| T-061-07-E | Elevation of Privilege | generic table vs XI/domain authority | mitigate | Table transforms only presentation rows; XI actions remain existing matchday callbacks keyed by player ID. |
| T-061-07-I | Information Disclosure | live/status copy | accept | Announcements contain localized column/view names and bounded reasons only; no paths, raw payloads, or private data are present. |
</threat_model>
