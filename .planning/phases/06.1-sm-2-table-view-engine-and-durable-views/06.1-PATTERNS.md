```rust
// crates/application/src/matchday.rs:34-64
pub struct MatchdayService<R> {
    repository: R,
}

impl<R: MatchdayRepository> MatchdayService<R> {
    pub fn new(repository: R) -> Self { Self { repository } }

    pub fn state(&self) -> Result<MatchdayState, MatchdayServiceError> {
        match self.repository.load().map_err(MatchdayServiceError::Persistence)? {
            Some(state) => Ok(state),
            None => {
                let state = MatchdayState::default();
                self.repository.save(&state).map_err(MatchdayServiceError::Persistence)?;
                Ok(state)
            }
        }
    }
}
```

```rust
// crates/application/src/matchday.rs:95-106
#[derive(Default)]
struct MemoryRepository {
    state: RefCell<Option<MatchdayState>>,
}

impl MatchdayRepository for MemoryRepository {
    fn load(&self) -> Result<Option<MatchdayState>, String> {
        Ok(self.state.borrow().clone())
    }
    fn save(&self, state: &MatchdayState) -> Result<(), String> {
        self.state.replace(Some(state.clone()));
        Ok(())
    }
}
```

**Copy:** injected generic port, typed application errors, load-or-seed behavior and in-memory service tests.  
**Extend:** table-view errors must distinguish invalid state, incompatible future schema, migration/quarantine recovery and unavailable persistence; do not collapse these into UI-authored strings.  
**Boundary:** application owns lifecycle/migration policy; it imports no Tauri or filesystem API.

## Shared Patterns

- TypeScript relative imports include `.js`; types use `import type`, readonly inputs and copied arrays at an IPC boundary.
- React components are controlled renderers; stateful authority stays in a screen controller or Rust service.
- Stable semantic IDs are data; localized Portuguese labels are presentation.
- Pure transforms clone input collections and preserve deterministic tie-breakers.
- Rust application ports are generic and injected; platform owns file I/O; Tauri owns host composition only.
- Application errors are typed before the host converts them to command results.
- User-facing recovery copy is Portuguese and actionable; diagnostics remain bounded and adapter-specific.
- Tests colocate with TypeScript/Rust modules; browser product behavior remains in `browser-tests/matchday.spec.ts`.
- No authentication/authorization guard pattern applies: this is a fixed local Tauri repository with no network endpoint.

## Planner Handoff

Plans should group work by authority, not by visual proximity:

1. pure engine/schema/tests;
2. Rust application/platform repository and Tauri/client boundary;
3. one-time legacy migration plus controlled screen integration;
4. saved-view/customizer UI and accessibility;
5. component/browser/full regression.

Do not place Rust persistence and React customization in one task, and do not advance to Phase 9 ownership while closing these gaps.
