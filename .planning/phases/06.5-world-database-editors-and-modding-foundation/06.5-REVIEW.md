# Fase 06.5 — Review único

## Protocolo

Foi utilizado exatamente um subagente revisor, somente após a implementação inicial. O revisor
não editou arquivos e não criou commits. Esta correção foi executada inline e não abriu outro ciclo
de revisão.

## Findings e resolução

| Prioridade | Finding | Resolução |
| --- | --- | --- |
| P1 | Exportação ignorava entrypoints arbitrários e não materializava assets. | World/patches agora são gravados nos entrypoints declarados. Como o editor JSON não transporta bytes, qualquer asset declarado ou patch de asset bloqueia antes de mutar o catálogo com `package.asset_payload_missing`. |
| P1 | Paths podiam atravessar symlink/junction; checksum de asset não era verificado; blacklist SVG era incompleta. | A raiz e cada arquivo existente são canonicalizados, cada componente é inspecionado contra symlink/reparse point, SHA-256 é comparado com os bytes exatos e todo SVG local é recusado no catálogo v1. |
| P1 | Pacote privado podia ser exportado ao catálogo público. | `packageId` privado e qualquer `visibility=privateDevelopment` são bloqueados na aplicação e novamente no repositório, antes de criar raiz, temporário, backup ou destino. |
| P1 | Frontend só conhecia assets compilados por `import.meta.glob`. | O catálogo ganhou `sourcePackageId` e `runtimeSource` controlados pelo backend; o frontend prefere essa fonte validada e mantém o glob apenas para assets oficiais compilados. A ativação/seleção de mods continua reservada à 06.6. |
| P2 | Schemas não eram aplicados diretamente; unknown fields e referências/ranges podiam passar. | Os novos contratos Rust usam `deny_unknown_fields`; o schema formal foi aprofundado; validação cobre bandeira, estádio, competição da temporada, função/posição, ranges, participantes duplicados e regras básicas. Testes negativos comprovam parsing e diagnósticos. |
| P2 | Matchday/Profile não compartilhavam o snapshot resolvido pelo coordenador mundial. | O startup resolve e valida uma vez pelo `WorldDatabaseCoordinator`, injeta `matchday` e `profiles` desse mesmo snapshot nos coordenadores e gerencia a mesma instância mundial. |
| P2 | Playwright modificou screenshots pertencentes à 06.4. | Os nove screenshots foram restaurados explicitamente ao `HEAD`; nenhum screenshot 06.4 aparece no diff final. |

## Evidências focadas

- 21 testes Rust filtrados da fundação mundial aprovados.
- 12 testes frontend/tooling focados aprovados.
- Guard de pacote público aprovado.
- `cargo check --workspace`, TypeScript e `git diff --check` aprovados após as correções.
- Checksum do pacote e SHA-256 de cada asset oficial verificados por teste.

## Resultado

Zero findings abertos. Nenhuma alteração 06.6+, nenhum redesign global e nenhuma representação
esportiva paralela foram introduzidos.
